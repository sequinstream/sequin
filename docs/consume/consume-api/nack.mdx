---
title: 'Nack messages'
description: "Nack messages for a Consume endpoint"
api: 'POST /http_pull_consumers/{consumer_name_or_id}/nack'
---

## Request parameters

<ParamField path="consumer_name_or_id" type="string" required>
  The ID or name of the consumer.
</ParamField>
<ParamField body="ack_ids" type="string[]" required>
  An array of `ack_id` values to nack.

  Each message returned by the [`/receive` endpoint](/consume/consume-api/receive) contains an `ack_id` that you can use to `nack`` the message.
</ParamField>

<RequestExample>
```shell
curl --request POST \
  --url https://api.sequinstream.com/api/http_pull_consumers/orders-pull-consumer/nack \
  --header 'Authorization: Bearer YOUR_API_TOKEN'\
  --header 'Content-Type: application/json' \
  --data '{
    "ack_ids": ["MTYyeJ7abUjl1pO", "MTYyeJ7abUjl1pP"]
  }'
```
</RequestExample>

<ResponseExample>
```json 204 OK
<No content>
```
</ResponseExample>
