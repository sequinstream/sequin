---
title: Metrics
description: Reference for Sequin's metrics. Use metrics to monitor health and performance.
---

Sequin exposes a metrics endpoint, using Prometheus text format, which can be integrated into your existing observability workflows.
By default, this endpoint is at `/metrics` on port 7377.

Metrics are designed to answer:
- Is Sequin keeping up with the database? (Ingestion metrics)
- Are sinks keeping up with Sequin? (Delivery metrics)
- Is the Sequin service experiencing CPU or memory pressure? (VM metrics)

## Metrics reference

### Ingestion metrics
- `sequin_ingestion_latency_ms` (Gauge)
  Measures latency between transaction commit and Sequin replication slot ingestion.
  Calculated by instrumenting Sequin's own heartbeat messages.
- `sequin_messages_ingested_count` (Counter)
  Number of messages Sequin has received on a given replication slot.

### Delivery metrics
- `sequin_delivery_latency_ms` (Histogram)
  Measures latency between Sequin and a given consumer's successful acknowledgement.
  Some consumers are able to batch messages, in which case this metric measures
  the batch latency, rather than the per-message latency.
- `sequin_message_deliver_attempt_count` (Counter)
  Incremented for each message Sequin tries to deliver, before receiving a success or failure response.
- `sequin_message_deliver_success_count` (Counter)
  Incremented for each message delivered and acknowledged.
- `sequin_message_deliver_failure_count` (Counter)
  Incremented for each message which was delivered but failed downstream.

### VM metrics
The BEAM VM exposes a many metrics - the following are the ones we find most useful for monitoring Sequin.
- `erlang_vm_statistics_reductions_total` (Counter)
  Measures how many VM instructions are executed across all processes.
  Shows intensity of CPU-bound work.
  Recommend using with a rate query rather than directly.
- `erlang_vm_memory_bytes_total` (Gauge)
  Memory usage of the entire VM.
